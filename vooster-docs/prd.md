## 제품 요구 사항 문서 (PRD)

### 1. Introduction

이 문서는 프리랜서가 플랫폼에 의존하지 않고,
직접적인 영업을 통해 잠재 고객에게 제안서를 작성·발송하며
클라이언트를 확보할 수 있도록 실행 과정을 돕는
시스템의 요구 사항을 설명합니다.

### 2. Goals

* 프리랜서가 플랫폼 외부에서 클라이언트를 확보할 수 있도록 지원
* 사용자가 최소 제안서를 단계별로 완성하고 실제 발송까지 도달하게 함
* 클라이언트 확보를 위한 체계적인 접근 방식 제공

### 3. Target Audience

경력 2년 미만의 프리랜서
최소 1건 이상의 유료 프로젝트 경험이 있는 프리랜서
플랫폼 기반 클라이언트 확보에서 벗어나고자 하는 프리랜서
제안서 작성 및 아웃바운드 영업에 익숙하지 않은 프리랜서

### 4. Key Features

***기업 기반 제안서 완성 플로우**
  - 선택된 기업 1곳을 기준으로 최소 제안서를 단계별로 완성하는 기능
  - 각 단계에서 사용자가 먼저 생각을 입력하고,
    AI는 해당 생각을 제안서 문장으로 정리하는 역할만 수행
  - 최종 결과는 “보내도 되는 수준”의 제안서 텍스트

***잠재고객 선별 및 확인 워크플로우**
  - 사용자가 직접 입력한 후보를 기준으로
    오늘 제안해도 무리하지 않은 대상인지 점검하는 과정
  - AI는 판단이 아닌 확인과 언어화 역할만 수행
  - 최종 선택 책임은 사용자에게 있음

***하루 단위 행동 기록**
  - 오늘의 제안 대상과 제안서 상태를 기록
  - 실제 발송 후 “보냈다” 행동을 남김
  - 분석이나 성과 관리가 아닌,
    하루의 행동 완료 여부만 표시


### 5. User Stories

- 프리랜서로서, 무엇부터 해야 할지 고민하지 않고 제안할 1곳에 맞는 제안 구조를 만들고 싶습니다.
- 프리랜서로서, 플랫폼에 의존하지 않고 제안할 수 있는 잠재 고객을 찾아내고 싶습니다.
- 프리랜서로서, 아무것도 안 한 하루를 반복하지 않도록 행동을 기록하고 싶습니다.


### 6. Release Criteria

사용자가 잠재 고객 선택 → 제안서 완성 → 실제 발송까지의
실행 과정을 최소 1회 이상 완료할 수 있을 것

### 7. Development Strategy

핵심 기능 중심의 최소 요건 제품(MVP) 개발
가설 검증과 반복적 개선에 중점을 둔 린 스타트업 방식 적용
### 8. Success Metrics

* 최소 1회 이상 제안서 발송까지 도달한 사용자 비율
* 제안서 완성 → 발송까지 이어진 전환율
* 고객 확보 사례 수 (정성)


### 9. Future Considerations

* 본 제품은 개인 프리랜서의 첫 제안 실행에 집중하며,
  자동화, CRM, 커뮤니티 기능은
  향후 별도 검증 후 고려할 수 있다.


### 10. Risks and Challenges

* 사용자가 제안서 작성 단계에서 이탈할 가능성
* 잠재고객 선별 기준에 대한 신뢰 형성
* 초기 성공 경험을 빠르게 만들어내고 성공 사례 창출
